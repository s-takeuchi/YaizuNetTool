3. 画面の説明

3.1 メインコンソールについて

StkFwを起動するとメインコンソールが表示されます。
メインコンソールはStkFwを使用した各種操作を行う上での中心的なユーザーインターフェースとなります。
メインコンソールを使用して行うことができる操作には，主に以下のものがあります。
  • 各種ファイル操作ダイアログの起動
  • 処理要素の追加／削除／関連線の追加／関連線の削除
  • 処理要素のプロパティ設定ダイアログの起動
  • スレッドコントローラの起動
  • 全スレッド一括開始／停止
  • Variable Controllerの起動
  • ワークスペースサイズの変更／グリッドの設定

以下の表示例で示すように，メインコンソールは，ウィンドウメニュー部分とワークスペース部分から構成されています。
ウィンドウメニュー部分では，各種ダイアログやコントローラを起動するためのメニューを提供します。 ワークスペース部分では，データフローを定義するための機能を提供します。

3.2 メインコンソールの操作

(1) 各種ファイル操作ダイアログの起動
メインコンソールのウィンドウメニューから[File]-[Open]，[File]-[Save As]メニューをクリックすると， StkFwのデータをファイルからの読み込む，またはStkFwのデータをファイルへの書き込むためのダイアログが起動します。 [File]-[New]メニューをクリックすると，現在のデータを捨て，新規にワークスペースを作成するためのダイアログ が起動します。

(2) 処理要素の追加／削除／関連線の追加／関連線の削除

StkFwはデータの受信，加工，送信を処理要素（アイコン）と処理要素間の関連線で表現したデータフロー として定義することができます。
  • ワークスペース上の空白部分を右クリックし，[処理要素の追加]メニューから処理要素を 追加することができます。
  • ワークスペース上のアイコンを右クリックし，[処理要素の削除]メニューで処理要素を削除することができます。
  • ワークスペース上の複数のアイコンを選択後，[関連線の追加]，[関連線の削除]メニューを選択することにより， 処理要素間の関連線（処理間関連線およびスレッド間関連線）を追加または削除することができます。

(3) 処理要素のプロパティ設定ダイアログの起動

ワークスペース上のアイコン（処理要素）をダブルクリックするか，右クリックして[Properties]メニューを 選択することにより，処理要素のプロパティ設定ダイアログを起動することができます。 各処理要素の動作の詳細は 6. 処理要素についてを参照してください。

(4) スレッドコントローラの起動

スレッドコントローラはデータフローを構成する個々の処理スレッドを実行するための制御ダイアログです。 メインコンソールのウィンドウメニュー[Run]-[Thread Controller]を選択すると，スレッドコントローラが起動します。 詳細は 7.2 スレッドコントローラを参照してください。

(5) 全スレッド一括開始／停止

メインコンソールのウィンドウメニュー[Run]-[Start all threads]ですべてのスレッドを一括で開始することができます。 また，[Run]-[Stop all threads]ですべてのスレッドを一括で停止することができます。
詳細は 7.1 全スレッド一括開始／停止を参照してください。

(6) Variable controllerの起動

Variable controllerを起動できます。詳細は 5. 変数管理機能を参照してください。

(7) ワークスペースサイズの変更／グリッドの設定

ワークスペース上の空白部分を右クリックし，[グリッド]-[On]，[グリッド]-[Off]でグリッド線の有効／無効を切り替えることができます。 また，ワークスペースの背景部分を右クリックし，[ワークスペースサイズ変更]を選択することでワークスペースサイズの変更 ダイアログが表示され，そこでワークスペースのサイズを変更することができます。
