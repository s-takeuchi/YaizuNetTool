stklib - ActorRequest


This class is used for the communication between ActorStatusElement and GameStatusElement.
ActorRequest mainly contains following parameters.
- 4 integer parameters
- 2 string parameters

ActorStatusElement can sets itsown status into the parameters of ActorRequest.
And ActorStatusElement can post the ActorRequest instance into GameStatusElement in order to inform the status of ActorStatusElement.

Function int GetRequestType()
Description Get the request type.
One of the following values can be returned.

ActorRequest::NORMAL (0)
ActorRequest::SYNCHRO (10)
ActorRequest::IGNORE_SAME_REQ_SAME_ACTOR (100)
ActorRequest::IGNORE_SAME_REQ (101)
ActorRequest::IGNORE_SAME_ACTOR (102)
ActorRequest::OVERWRITE_SAME_REQ_SAME_ACTOR (200)
ActorRequest::OVERWRITE_SAME_REQ (201)
ActorRequest::OVERWRITE_SAME_ACTOR (202)
In
Out int: Request type

Function void SetRequestType(int)
Description Set the request type.
One of the following values can be set.

ActorRequest::NORMAL (0)
Normal Request

ActorRequest::IGNORE_SAME_REQ_SAME_ACTOR (100)
The request which has same request ID and same actor ID can not be registered into GameStatusElement.

ActorRequest::IGNORE_SAME_REQ (101)
The request which has same request ID can not be registered into GameStatusElement.

ActorRequest::IGNORE_SAME_ACTOR (102)
The request which has same actor ID can not be registered into GameStatusElement.

ActorRequest::OVERWRITE_SAME_REQ_SAME_ACTOR (200)
The request is overwritten on the existing request which has same request ID and same actor ID in GameStatusElement.

ActorRequest::OVERWRITE_SAME_REQ (201)
The request is overwritten on the existing request which has same request ID in GameStatusElement.

ActorRequest::OVERWRITE_SAME_ACTOR (202)
The request is overwritten on the existing request which has same actor ID in GameStatusElement.
In int: Request type
Out void
